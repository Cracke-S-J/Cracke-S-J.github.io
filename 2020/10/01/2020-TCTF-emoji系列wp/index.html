2020 TCTF quals eeeeeemoji

这题就是pwn了，看起来跟上题长的差不多，但是这时候我们再下断看寄存器可以发现很多寄存器都被设成0xdeafbeefdeadbeef了，现在只剩rdx、rdp、rip，rip不好动，我们可以and esp, edx，这时候会得到一个较小的值，这个值可能指向我们可控的地址。我们可以先mmap大量内存，布置好栈和shellcode，等到and esp, edx后，程序会执行add rsp, 0x8000，此时有概率会跳到我们布置好的地址上，这时候我们就可以控制栈去执行shellcode了。

2020 TCTF final Emoji Revenge

这题看起来还是跟初赛那两道题差不多：

  • 🍺 maps and prints an address of a random RWX page in 0x0 - 0x3e8000 range. Nothing stops us from calling it multiple times.
  • 🐮 puts the following code into 🍺-allocated page and jumps to it.
  • 🐴 is like 🐮, but lets us input 0x208 bytes of data. The bytes [0x100:0x200] are then overwritten by ‘a’s. The bytes [0x202:0x208] are overwritten by nops. So we control [0x0:0x100] and [0x200:0x202].

不过这题的2字节不是要跳到shellcode，而是要死循环，触发超时，让pc跳到0，然后mmap到0地址处。具体没看，要看那个洞。
